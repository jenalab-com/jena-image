# 이미지 북마크(즐겨찾기) 기능 설계

작성일: 2026-06-20

## 목적

폴더 구조와 무관하게, 자주 보거나 다시 찾고 싶은 이미지를 **위치만 저장**해
한곳(사이드바의 북마크 항목)에서 썸네일로 모아 본다. 이미지가 여기저기 흩어져
있어 폴더를 일일이 열어 찾기 어려운 문제를 해결한다.

> 용어 주의: 이 "북마크"는 즐겨찾기다. 기존 `SecurityScopeService`의
> security-scoped bookmark(폴더 접근 권한 영속화)와는 다른 개념이다.

## 핵심 결정 (확정)

| 항목 | 결정 |
|---|---|
| 저장 내용 | 파일 경로 목록(위치만). 원본을 복사하지 않음 |
| 저장 방식 | `UserDefaults`에 경로 문자열 배열 — 비샌드박스 앱이라 보안 스코프 북마크 불필요 |
| 위치 | 사이드바 최상단 "★ 북마크" 고정 항목 |
| 그리드 | 기존 `BrowserViewController`에 "북마크 모드" 추가(재사용) |
| 추가 진입 | 브라우저 + 뷰어 양쪽, **토글**(다시 누르면 해제) |
| 뷰 동작 | 열기 / 빼기(목록에서만) / 삭제(실제 파일·휴지통·확인) / 이름변경(실제 파일명) |
| 정렬 | 최근 추가가 위 |
| 깨진 북마크 | 플레이스홀더로 표시, 자동 삭제 안 함 |
| 드래그 재정렬 | 2차로 미룸 |

## 구성 요소

### 1. `BookmarkStore` (신규)
북마크 = 파일 URL의 순서 있는 목록. `UserDefaults`에 경로 문자열 배열로 영속.
- `var bookmarks: [URL]` — 최근 추가가 앞(index 0).
- `func add(_ url: URL)` — 이미 있으면 무시(중복 방지). 새 항목을 맨 앞에.
- `func remove(_ url: URL)` — 목록에서만 제거(파일은 그대로).
- `func contains(_ url: URL) -> Bool` — 토글·★ 상태 표시용.
- `func rename(from: URL, to: URL)` — 이름변경 시 경로 교체(순서 유지).
- 변경 시 `NotificationCenter`로 `.bookmarksChanged` 발행 → 사이드바·그리드·뷰어 버튼 갱신.
- 단일 인스턴스를 `MainWindowController`가 소유해 주입(기존 `securityService` 패턴과 동일).

### 2. 사이드바 — "★ 북마크" 항목
폴더 트리(`NSOutlineView`) 위에 고정 행 하나.
- `SidebarDelegate`에 `sidebarDidSelectBookmarks(_:)` 추가.
- 클릭 → `MainWindowController`가 메인 영역을 북마크 그리드로 전환. 폴더를 다시
  선택하면 일반 폴더 보기로 복귀.
- 구현 메모: 현재 사이드바는 폴더 루트 배열을 outline에 그린다. 북마크 행은
  트리 데이터가 아니라 별도 최상단 고정 항목으로 다룬다(폴더와 구분).

### 3. 북마크 그리드 — `BrowserViewController` 북마크 모드
북마크는 여러 폴더에 흩어진 이미지의 평면 목록(폴더 없음).
- `func displayBookmarks(_ files: [ImageFile])` — 임의 이미지 배열을 직접 그린다.
  기존 그리드·썸네일·선택·키보드 로직을 그대로 재사용.
- 모드 플래그(`isBookmarkMode`)로 폴더 더블클릭 진입을 비활성, 우클릭 메뉴에
  "빼기"를 추가, 빈 상태 안내("북마크가 비어 있어요")를 표시.
- 북마크 목록 → `ImageFile`로 변환 시 파일 존재를 확인(아래 6번).

### 4. 추가/제거 (토글)
- **브라우저**: 선택 → 우클릭 "북마크에 추가/빼기"(상태에 따라 라벨 토글) + 단축키.
  메뉴 항목은 기존 "비교" 옆에 추가.
- **뷰어**: 현재 이미지 → View 메뉴 항목 + 단축키 + 하단/툴바 ★ 버튼. 이미
  북마크면 ★ 채워짐, 아니면 빈 ★. 누르면 토글.
- 단축키는 기존과 충돌하지 않는 키로 배정(구현 단계에서 확정).

### 5. 북마크 뷰에서의 동작
- **열기**: 더블클릭 → 뷰어. 이때 imageList는 북마크 목록 전체(strip 네비가 북마크 안에서 이동).
- **빼기**: 우클릭 "빼기" → `BookmarkStore.remove`. 파일은 그대로.
- **삭제**: 우클릭 "삭제" → 기존 `performDelete`(휴지통·확인 다이얼로그) 재사용.
  삭제 성공 시 `BookmarkStore`에서도 제거.
- **이름변경**: 기존 `performRename` 재사용. 성공 시 `BookmarkStore.rename(from:to:)`로
  경로 갱신.

### 6. 깨진 북마크 처리
원본이 외부에서 이동·삭제되면 경로가 무효가 된다.
- 북마크 그리드를 채울 때 각 경로의 파일 존재를 확인.
- 없는 항목은 흐릿한 플레이스홀더 + "원본을 찾을 수 없음" 표시. "빼기"만 가능.
- **자동 삭제하지 않는다**(잠깐 옮겼다 되돌릴 수 있으므로). 사용자가 직접 빼기.

## 범위 (1차)

포함:
- `BookmarkStore`(추가/빼기/포함/이름변경 + 영속 + 변경 알림)
- 사이드바 "★ 북마크" 항목 + 그리드 전환
- `BrowserViewController` 북마크 모드(평면 목록 표시 + 빈 상태 + 빼기 메뉴)
- 브라우저·뷰어 양쪽 토글 추가(메뉴·단축키·★ 버튼·우클릭)
- 북마크 뷰 동작: 열기·빼기·삭제(파일)·이름변경(파일), Store 동기화
- 깨진 북마크 플레이스홀더

제외(2차 이후):
- 드래그 재정렬
- 북마크 그룹/폴더, 태그, 검색
- 북마크 표시용 별칭(이름변경은 실제 파일명만)

## 열린 사항(구현 단계에서 확정)
- 북마크 토글 단축키 배정(기존 단축키와 무충돌 확인).
- 사이드바 고정 항목을 outline 헤더로 둘지 별도 작은 뷰로 둘지.
- 깨진 북마크 플레이스홀더의 시각 처리(아이콘/회색 썸네일).
