# 이미지 비교 기능 설계

작성일: 2026-06-20

## 목적

여러 후보 사진 중 베스트를 고르기 위한 **나란히 비교(side-by-side)** 기능.
2~4장을 동시에 띄우고 **줌·팬을 동기화**해 같은 위치를 같은 배율로 들여다보며
어느 사진이 더 선명한지·구도가 좋은지 판단한다.

> 같은 이미지의 전/후 차이(overlay·wipe·diff)는 이번 범위가 아니다. 2차 과제로 분리.

### 참고 — 다른 뷰어의 방식
- **FastStone**: 다중 선택 후 비교, 최대 4장 그리드, 줌·팬 동기화
- **XnView MP / ACDSee**: 다중 선택 → 나란히, 배율 잠금 + 동기 패닝
- 공통점: ① 칸마다 리스트/불러오기 버튼을 두지 않고 *브라우저에서 미리 다중 선택해 진입*,
  ② **줌·팬 동기화가 비교의 핵심**(나란히 띄우는 것만으로는 부족).

## 사용자 흐름

1. 브라우저에서 사진을 2장 이상 다중 선택(Cmd/Shift+클릭 — 이미 지원).
2. "비교" 실행(메뉴 + 단축키, 가능하면 우클릭 컨텍스트 메뉴).
3. 별도 비교 창이 열리고 선택분이 그리드로 배치된다.
4. 한 칸에서 확대/이동하면 모든 칸이 같은 배율·위치로 따라간다(동기화 토글로 끌 수 있음).
5. 칸을 클릭해 "활성 칸"으로 만든 뒤, 하단 공용 strip에서 다른 사진을 클릭하면 그 칸이 교체된다.
6. 칸의 닫기(×)를 누르면 그 칸이 빠진다(4→3→2). 빈 칸에는 strip에서 채워 넣을 수 있다.
7. ESC 또는 창 닫기로 종료.

## 구성 요소

### 1. 진입점
- `BrowserViewController.selectedURLs()`로 선택분을 얻는다(이미 존재).
- 2장 이상일 때만 "비교" 활성화. 1장 이하면 비활성.
- 트리거 4종(모두 같은 액션으로 연결):
  - **MainMenu 항목 + 단축키**
  - **브라우저 우클릭 컨텍스트 메뉴**
  - **상단 툴바 아이콘** — 기존 `NSToolbar`에 `ToolbarID.compare` 아이템 추가
    (`MainWindowController+Delegates.swift`의 default/allowed 식별자와 itemForItemIdentifier에 등록).
    선택 수가 2 미만이면 `validateToolbarItem`/액션 검증으로 비활성 처리.
- 선택분에서 비디오는 제외(정지 이미지 비교 전용). 제외 후 2장 미만이면 비활성.

### 2. `CompareWindowController` (별도 윈도우)
- 기존 `ImageEditorWindowController`와 같은 패턴으로 윈도우 생성·생명주기 관리.
- 선택 수에 따라 자동 레이아웃:
  - 2장 → 좌우 1×2
  - 3장 → 가로 1×3
  - 4장 → 2×2
  - 레이아웃은 `NSGridView`(또는 `NSStackView` 중첩)로 구성하고, 칸 추가/제거 시 재배치.
- 상단 툴바: **동기화 on/off 토글**, (선택) 전체 맞춤(fit) 버튼.
- 하단: 공용 후보 썸네일 strip(아래 4번).

### 3. `ComparePaneView` (개별 칸)
- 줌·팬 가능한 이미지 + 파일명 라벨 + 닫기(×) 버튼.
- 이미지 표시는 기존 `ImageDisplayView`를 **재사용**한다(줌·팬·magnification·fit 로직이 이미 있음).
  - 단, 현재 `ImageDisplayView`는 줌/스크롤 변경을 외부에 알리지 않는다.
    → **줌·팬이 바뀔 때 호출되는 콜백 하나를 추가**한다(예: `onTransformChanged: ((magnification, visibleRect) -> Void)`).
  - 편집/flip/rotate 등 비교에 불필요한 기능은 호출하지 않을 뿐, 코드는 공유한다.
- 클릭하면 자신이 "활성 칸"이 되었음을 컨트롤러에 알린다(테두리 하이라이트).

### 4. 하단 공용 후보 strip
- 후보 풀 = 처음 선택한 사진들 + 같은 폴더의 나머지 이미지.
- 가로 썸네일 목록. 항목 클릭 → 현재 활성 칸에 로드(교체).
- 기존 `ThumbnailStripView`는 세로 전용이라 그대로 못 쓴다. 가로용 경량 뷰를 새로 두거나
  `ThumbnailStripView`를 방향 파라미터로 일반화한다(구현 단계에서 택일).

### 5. `SyncCoordinator` (동기화)
- 각 `ComparePaneView`의 `onTransformChanged` 신호를 받아 나머지 칸에 같은 변환을 적용한다.
- 피드백 루프(브로드캐스트가 다시 콜백을 트리거)는 `isSyncing` 플래그로 차단.
- 토글이 off면 브로드캐스트하지 않아 칸마다 독립 조작 가능.
- 좌표 기준: 각 이미지는 크기가 다를 수 있으므로 **정규화된 중심점 + 배율**로 동기화
  (절대 픽셀이 아니라 "이미지의 어느 지점을 어느 배율로"). 크기가 크게 다르면 완벽히
  일치하진 않으나 실사용엔 충분 — 구현 단계에서 정밀도 확인.

## 범위 (1차)

포함:
- 브라우저 다중 선택 → 비교 진입(메뉴 + 단축키)
- `CompareWindowController` + 2~4 가변 그리드
- `ComparePaneView`(`ImageDisplayView` 재사용 + 변환 콜백 추가)
- `SyncCoordinator`(줌·팬 동기화 + on/off 토글)
- 하단 공용 strip + 활성 칸 클릭 교체
- 칸 닫기(제거)

제외(2차 이후):
- 드래그&드롭 교체
- 전/후 차이 보기(overlay·wipe·diff)
- 1:1 픽셀 정밀 정렬 등 고급 동기화

## 열린 사항(구현 단계에서 확정)
- 그리드 구성을 `NSGridView`로 갈지 `NSStackView` 중첩으로 갈지.
- 하단 strip을 새 가로 뷰로 만들지 `ThumbnailStripView` 일반화로 갈지.
- 동기화 좌표 정규화의 정밀도(이미지 크기 차가 클 때).
