# 2026-04-10 개발 로그

## 외부 편집 시 이미지 즉시 반영 (Photoshop 등)

### 요청
이미지를 보는 중에 Photoshop에서 동일 이미지를 수정하고 저장해도 뷰어에 반영되지 않음.

### 원인
1. `FolderWatcher`는 폴더의 `.write/.rename/.delete` 이벤트를 감지하고 있었으나
   `MainWindowController.refreshAfterExternalChange()`가 `contentMode == .browser`일 때만
   화면을 갱신했다. 뷰어 모드에서는 무시되었다.
2. `ViewerViewController`에는 현재 표시 중인 이미지를 디스크에서 다시 읽는
   API 자체가 없었다.
3. 썸네일 스트립도 `ThumbnailCache`에 캐싱된 이전 이미지를 그대로 사용한다.

### 변경 사항
- [Sources/viewer/ViewerViewController.swift](Sources/viewer/ViewerViewController.swift)
  `reloadCurrentImage()` 추가. ThumbnailCache invalidate → 썸네일 행 reload →
  `showImage(file)` 재호출 순으로 동작.
- [Sources/viewer/ThumbnailStripView.swift](Sources/viewer/ThumbnailStripView.swift)
  `reloadThumbnail(at:)` 추가. 특정 row만 다시 그린다.
- [Sources/app/MainWindowController.swift](Sources/app/MainWindowController.swift)
  `refreshAfterExternalChange()`에서 `contentMode == .viewer` 분기를 추가하여
  `viewerVC.reloadCurrentImage()` 호출.

### 동작 흐름
1. Photoshop이 이미지 저장 → 파일시스템 mtime 변경
2. `FolderWatcher`의 `DispatchSourceFileSystemObject`가 부모 폴더의
   `.write` 이벤트 감지
3. 0.3초 디바운스 후 `handleFolderChange` → `refreshAfterExternalChange`
4. 뷰어 모드면 `viewerVC.reloadCurrentImage()` 호출
5. `CGImageSourceCreateWithURL`은 캐시 없이 디스크에서 새로 읽으므로
   변경된 픽셀이 즉시 반영됨

### 검증
- `make build` 통과.
- 수동 테스트 필요: 이미지를 뷰어로 연 상태에서 Photoshop에서 동일 파일 저장 →
  0.3초 이내 새 픽셀로 갱신되는지 확인.

### 트레이드오프
- 디바운스 0.3초 때문에 저장 직후 약간의 지연이 있다. Photoshop이 임시 파일로
  쓰고 rename하는 패턴이라 이벤트가 여러 번 발생할 수 있어 디바운스는 유지.
- 같은 폴더의 다른 파일이 변경되어도 현재 보고 있는 이미지가 reload된다.
  파일 수정 시각(mtime) 비교로 최적화할 수 있지만, reload는 디스크 I/O
  한 번이라 비용이 작아 일단 단순화 우선으로 처리.
