# 2026-06-21 — 북마크 뷰 동작 완성 (Task 4)

## 배경

Tasks 1–3에서 `BookmarkStore`, 사이드바 "★ 북마크" 진입, 브라우저·뷰어 토글까지 구현됐다.
Task 4는 북마크 그리드의 우클릭 동작(열기/빼기/이름변경/삭제), 파일 조작 시 Store 동기화,
더블클릭 viewer 진입 시 북마크 목록 네비게이션, 깨진 북마크 플레이스홀더를 완성한다.

## 변경 사항

### `Sources/browser/BrowserViewController.swift`

- `BrowserDelegate`에 `browserDidRequestRemoveBookmark(_:urls:)` 추가.
- `menuNeedsUpdate`에 북마크 모드 전용 분기 추가:
  - 선택 URL 중 파일이 존재하지 않는(깨진) 항목이 하나라도 있으면 열기/이름변경/삭제를 `nil` action으로 비활성화.
  - "빼기"는 항상 활성 (`contextRemoveBookmark`).
  - 다중 선택 시 "이름 변경" 미표시.
  - 북마크 모드 메뉴 후 `return` — 일반 메뉴 항목(복사/이동/폴더색상 등) 미표시.
- `contextRemoveBookmark(_:)` 핸들러 추가 → `delegate?.browserDidRequestRemoveBookmark`.
- `contextOpen(_:)`: 깨진 파일(fileExists false)이면 early return; 북마크 모드에서는 `contents` 기반 목록 전달.
- `handleDoubleClick(at:)`: 북마크 모드에서 `contents`에서 이미지만 추출해 viewer의 `inList`로 전달 (↑/↓가 북마크 목록 내 이동). 깨진 북마크는 열기 차단.

### `Sources/browser/BrowserItem.swift`

- `configure(with:thumbnail:)` `.image` 케이스에서 `FileManager.default.fileExists` 검사:
  - 파일 없음 → `exclamationmark.triangle` 시스템 심볼 + `tertiaryLabelColor`(흐릿함) + 이름 레이블도 회색.
  - 자동 제거 없음 — 플레이스홀더로만 표시.

### `Sources/app/MainWindowController+FileOperations.swift`

- `performDelete(urls:)`: 휴지통 이동 성공 경로에 `bookmarkStore.remove(url)` 추가.
- `performRename(url:newName:)`: `.success(let newURL)` 경로에 `bookmarkStore.rename(from: url, to: newURL)` 추가 (newURL은 기존 코드가 이미 계산).

### `Sources/app/MainWindowController+Delegates.swift`

- `BrowserDelegate` 구현에 `browserDidRequestRemoveBookmark(_:urls:)` 추가:
  `urls.forEach { bookmarkStore.remove($0) }`.

## 결정 사항

- **"Finder에서 보기" 북마크 메뉴에서 제외**: `BrowserViewController`에 `contextRevealInFinder` 핸들러가 없고, `MainWindowController`의 reveal 경로는 메뉴바 액션(별도 체인)이라 북마크 컨텍스트 메뉴에서 호출하기 어렵다. YAGNI 원칙에 따라 이번 범위에서 제외.

## 수동 GUI 검증 체크리스트

1. 북마크 그리드 우클릭 → 열기/빼기/이름 변경/삭제 표시 확인
2. **빼기** → 북마크에서만 제거, 파일은 Finder에 그대로 존재
3. **이름 변경** → 실제 파일명 변경 + 북마크 그리드에 새 이름으로 갱신
4. **삭제** → 확인 후 휴지통 이동 + 북마크에서도 제거
5. 북마크 이미지 더블클릭 → 뷰어에서 ↑/↓가 북마크 목록 내 이동
6. 북마크 파일을 Finder에서 다른 곳으로 이동 후 그리드 → 경고 심볼 플레이스홀더, 빼기만 활성
