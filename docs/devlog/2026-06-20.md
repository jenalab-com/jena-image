# 2026-06-20 개발 로그

## 비교 창 — 하단 후보 strip + 활성 칸 교체 (Task 4)

### 배경
Tasks 1–3에서 만든 비교 창(2–4 칸 그리드, 줌·팬 동기화, 칸 닫기)에
하단 후보 썸네일 strip과 활성 칸 교체 기능을 추가했다.

### 변경 사항

**`Sources/viewer/ImageDisplayView.swift`**
- `var onSingleClick: (() -> Void)?` 추가.
- `mouseDown(with:)`에서 더블클릭 분기 이후, 패닝 시작 전에 `onSingleClick?()` 호출.
  → 이미지 영역 클릭도 부모 뷰 활성화 콜백까지 전파되도록 함.

**`Sources/viewer/ComparePaneView.swift`**
- `var onActivated: ((ComparePaneView) -> Void)?` 추가.
- `setActive(_ active: Bool)` — 강조 테두리(accent 3pt) / 기본 테두리(separator 1pt) 토글.
- `setFile(_ file: ImageFile)` — 표시 이미지를 다른 파일로 교체 후 `load()` 재호출.
- `mouseDown(with:)` 오버라이드 — 여백 영역 클릭 시 `onActivated` 전달.
- `setupViews()`에서 `imageDisplayView.onSingleClick` 연결 — 이미지 영역 클릭도 활성화.

**`Sources/viewer/CompareCandidateStrip.swift`** (신규)
- `NSScrollView` + `NSStackView` + `NSButton` 조합으로 가로 썸네일 목록 구현.
- `ThumbnailCache.shared`에서 캐시 확인, 없으면 `ImageServiceProtocol.generateThumbnail`로 비동기 로드.
- 클릭 시 `onSelect: ((ImageFile) -> Void)?` 호출.

**`Sources/viewer/CompareWindowController.swift`**
- `private weak var activePane`, `private var candidateStrip` 추가.
- `setupGrid()` 레이아웃을 [토글]→[그리드]→[strip(고정 70pt)] 3단으로 확장.
- `buildCandidates(from:)` — 첫 파일의 부모 폴더를 `FileService().contentsOfFolder(at:sortKey:ascending:)`로 열거 후, 선택 파일 + 폴더 이미지를 URL 기준 중복 제거·비디오 제외.
- `setActivePane(_:)` — 이전 활성 칸 비활성화 후 새 칸 활성화.
- `handleCandidateSelected(_:)` — 활성 칸(없으면 첫 칸)의 이미지를 strip 선택 파일로 교체.
- `removePane(_:)`에서 닫힌 칸이 활성 칸이면 첫 칸으로 활성 이동.
- `init`에서 각 pane에 `onActivated` 콜백 연결, 초기 활성 칸 = 첫 칸.

### 설계 판단
- `ImageDisplayView`가 자체 `mouseDown`으로 패닝을 처리하므로, 오버레이 없이
  `onSingleClick` 훅을 끼워 ComparePaneView까지 전파하는 방식을 택했다.
- 줌·팬 동기화는 `imageDisplayView` 인스턴스가 그대로 유지되므로 `setFile` 후
  `CompareSyncCoordinator`를 재배선할 필요가 없다.

### 수동 GUI 검증 체크리스트
1. 같은 폴더 이미지가 여럿인 상태에서 2장 비교 → 하단 strip에 폴더 이미지들이 표시되는지.
2. 칸 클릭 → 강조 테두리(accent)가 생기는지. 다른 칸 클릭 시 활성이 이동하는지.
3. 활성 칸이 있는 상태에서 strip의 다른 이미지 클릭 → 활성 칸 이미지가 교체되는지.
4. 교체 후 그 칸도 줌·팬 동기화에 정상 참여하는지.
5. 칸을 닫아 2칸이 된 뒤에도 strip 교체가 동작하는지.

## 뷰어 우측 썸네일 strip — 긴 파일명 다중 줄 표시

### 배경
이미지 크게 보기(뷰어)의 우측 세로 썸네일 목록에서 파일명이 길면
한 줄에 가운데 잘림(`.byTruncatingMiddle`, `maximumNumberOfLines = 1`)으로
욱여넣어져 깨져 보였다.

### 변경 사항
[Sources/viewer/ThumbnailStripView.swift](../../Sources/viewer/ThumbnailStripView.swift)
- 셀 텍스트필드를 멀티라인으로 전환: `maximumNumberOfLines = 3`,
  `lineBreakMode = .byTruncatingTail`, `usesSingleLineMode = false`, `cell?.wraps = true`.
  → 이름이 길면 1→2→3줄로 자연스럽게 줄바꿈, 3줄 초과분은 말줄임(`…`).
- 고정 `rowHeight`(76) 제거, `tableView(_:heightOfRow:)` 동적 계산으로 교체.
  - `nameHeight(for:)`가 현재 셀 너비에서 이름을 그렸을 때의 줄 수(1~3 clamp)를
    `boundingRect`로 계산. 한 줄 기준 높이도 동일 방식("X" 측정)으로 구해 줄 수가
    깔끔하게 떨어지게 함.
  - row 높이 = 썸네일(60) + 상하 패딩(10) + 이름 높이.
- 우측 strip은 splitView 안에서 폭이 120~180으로 가변이라, 폭이 바뀌면 줄 수도
  달라진다. `layout()` 오버라이드에서 마지막 계산 너비와 비교해 변하면
  `noteHeightOfRows`로 전체 행 높이 재계산.

### 설계 판단
- 고정 높이로 항상 3줄 공간을 비워두는 대신 **가변 높이**를 택했다. 짧은 이름은 1줄로
  컴팩트하게 두어 한 화면에 보이는 썸네일 수를 유지하고, 긴 이름만 늘어나게 했다.

### 검증
- `make build` 성공(경고·에러 0). 동작 검증(GUI 수동 테스트)은 별도 필요.
