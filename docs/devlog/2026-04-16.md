# 2026-04-16 개발 로그

## 이름 변경 중 Cmd+C/V가 텍스트 대신 파일을 복사/붙여넣기하는 문제 수정

### 요청
파일 이름 변경 중 텍스트를 복사(Cmd+C)하면 텍스트가 아닌 선택된 이미지 파일이
클립보드에 복사됨. 붙여넣기(Cmd+V)도 마찬가지.

### 원인
Edit 메뉴의 Cmd+C가 `copyFiles(_:)`, Cmd+V가 `pasteFiles(_:)`에 직접 바인딩되어 있어,
텍스트 필드의 field editor(NSTextView)가 first responder인 상태에서도
파일 복사/붙여넣기 동작이 실행되었다.

### 변경 사항
- [Sources/app/MainWindowController.swift](Sources/app/MainWindowController.swift)
  - `copyFiles(_:)`: first responder가 field editor NSTextView이면 `textView.copy(sender)`로 위임
  - `pasteFiles(_:)`: 동일하게 `textView.paste(sender)`로 위임
  - `validateMenuItem(_:)`: 텍스트 편집 중이면 copy/paste 메뉴 항상 활성화

### 판단 근거
- AppKit에서 field editor는 NSTextView이며, `isFieldEditor == true`로 식별
- 이름 변경 시 사용되는 NSTextField는 내부적으로 field editor를 first responder로 사용
- field editor 여부만 체크하므로, 일반 텍스트 편집 상황(검색 등)에서도 올바르게 동작

## 장시간 방치 시 무작위 선택 영역 발생 수정

### 요청
프로그램을 오래 켜놓으면 2~3개 항목이 무작위로 선택되는 현상.

### 원인
1. `FolderWatcher`가 macOS 백그라운드 작업(.DS_Store 갱신, Spotlight 인덱싱 등)으로
   주기적으로 트리거됨
2. `refreshAfterExternalChange()`에서 `browserVC.display()` → `collectionView.reloadData()` 호출
3. NSCollectionView의 `reloadData()`는 **selection index path를 유지**하지만,
   폴더 내용이 변경되면 동일한 인덱스에 다른 아이템이 위치하게 됨
4. 결과적으로 사용자가 선택하지 않은 항목이 선택된 것처럼 보임

### 변경 사항
- [Sources/browser/BrowserViewController.swift](Sources/browser/BrowserViewController.swift)
  - `display()` 내 `reloadData()` 직후 `deselectAll(nil)` 추가하여 선택 초기화
  - `restoreSelection(urls:)` 추가 — URL 기반으로 올바른 인덱스에 선택 복원
- [Sources/app/MainWindowController.swift](Sources/app/MainWindowController.swift)
  - `refreshAfterExternalChange()`에서 선택을 URL로 저장 → display → URL로 복원

### 판단 근거
- 인덱스 기반 선택 보존은 내용이 바뀌면 틀린 항목을 가리킴
- URL 기반으로 보존/복원하면 항목 순서가 바뀌어도 올바른 항목이 선택됨
- `display()` 자체에서 `deselectAll`을 호출하여, refresh 외 경로(네비게이션)에서도
  고스트 선택이 발생하지 않도록 방어
