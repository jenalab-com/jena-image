# JenaImage — 상세 기획안 (PRD)

## 1. 목적

macOS 사용자를 위한 네이티브 이미지 뷰어 애플리케이션을 개발한다. 로컬 파일 시스템의 폴더 구조를 탐색하며 이미지를 빠르게 브라우징하고, 뷰어에서 확대/축소하며, 다양한 포맷으로 변환·내보내기할 수 있는 도구를 제공한다.

---

## 2. 문제 정의

macOS 기본 미리보기(Preview)는 단일 이미지 열람에는 적합하나, 폴더 단위로 이미지를 탐색하며 정리(이동/복사/삭제)하고 포맷 변환까지 수행하려면 Finder ↔ Preview ↔ 변환 도구를 오가야 하는 비효율이 있다.

**JenaImage는 하나의 창 안에서 폴더 탐색 → 이미지 브라우징 → 뷰잉 → 파일 관리 → 포맷 변환을 모두 처리한다.**

---

## 3. 목표 / KPI

| 목표 | 측정 지표 |
|------|----------|
| 이미지 탐색 효율 | 폴더 선택 → 이미지 열람까지 2클릭 이내 |
| 포맷 변환 접근성 | 이미지 뷰 → 다른 이름으로 저장까지 2클릭 이내 |
| 파일 관리 효율 | 드래그 앤 드롭으로 이미지 이동 완료 |
| 앱 기동 속도 | 콜드 스타트 2초 이내 |

## 4. 대상 사용자 및 시나리오

### 대상 사용자
- macOS 사용자 중 로컬에 다수의 이미지를 폴더별로 관리하는 사람
- 디자이너, 사진작가, 콘텐츠 제작자 등 이미지 파일을 빈번히 탐색·정리하는 사람

### 핵심 시나리오
1. **이미지 브라우징**: 사이드바에서 폴더를 선택하고 우측 브라우저에서 이미지를 썸네일로 훑어본다.
2. **이미지 뷰잉**: 원하는 이미지를 더블 클릭하여 확대 뷰어에서 상세 확인한다.
3. **파일 정리**: 여러 이미지를 선택하여 다른 폴더로 드래그해 이동하거나, 복사/삭제한다.
4. **포맷 변환 후 저장**: 이미지를 원하는 포맷(jpg, png, webp 등)으로 변환하여 다른 이름으로 저장한다.

## 5. 범위 (In-Scope)

| # | 기능 |
|---|------|
| F1 | 좌측 사이드바 — 폴더 트리 구조 표시 및 탐색 |
| F2 | 우측 브라우저 — 선택 폴더의 하위 폴더 + 이미지 목록 표시 |
| F3 | 이미지 뷰어 — 우측 패널 내 인라인 뷰어 (별도 창 아님) |
| F4 | 이미지 뷰어 — 좌측 썸네일 스트립 + 우측 이미지 뷰 |
| F5 | 이미지 확대/축소 |
| F6 | 다른 이름으로 저장 — 포맷 변환 지원 (jpg, png, webp, heic, heif, avif, tiff, bmp, gif) |
| F7 | 이미지 삭제 |
| F8 | 이미지 이동 (드래그 앤 드롭으로 폴더 지정) |
| F9 | 이미지 복사 |
| F10 | 파일 이름 변경 |

## 7. 기능 요구사항

### F1. 사이드바 — 폴더 트리

| 항목 | 설명 |
|------|------|
| 트리 구조 | 사용자 홈 디렉터리 또는 지정 루트 폴더부터 하위 폴더를 트리로 표시 |
| 펼침/접기 | disclosure triangle로 하위 폴더 펼침/접기 |
| 폴더 선택 | 클릭 시 우측 브라우저에 해당 폴더 내용 표시 |
| 드래그 타겟 | 이미지를 사이드바 폴더에 드롭하면 이동 |
| 비어있는 폴더 | 하위 폴더가 없는 경우 disclosure triangle 미표시 |

### F2. 브라우저 — 폴더/이미지 목록

| 항목 | 설명 |
|------|------|
| 표시 항목 | 현재 폴더의 하위 폴더 + 이미지 파일 |
| 폴더 표시 | 폴더 아이콘 + 폴더 이름 |
| 이미지 표시 | 썸네일 + 파일 이름 |
| 폴더 클릭 | 해당 폴더로 이동 (사이드바 선택도 연동) |
| 이미지 단일 클릭 | 이미지 선택 (하이라이트) |
| 이미지 더블 클릭 | 이미지 뷰어 모드 진입 |
| 정렬 | 폴더 먼저, 이미지 다음 (이름 기준 오름차순) |
| 지원 포맷 | jpg, jpeg, png, gif, bmp, tiff, webp, heic, heif, avif |
| 다중 선택 | Cmd+클릭(개별), Shift+클릭(범위) |
| 컨텍스트 메뉴 | 우클릭 시: 열기, 이름 변경, 복사, 이동, 삭제, 다른 이름으로 저장 |

### F3–F4. 이미지 뷰어

| 항목 | 설명 |
|------|------|
| 진입 | 브라우저에서 이미지 더블 클릭 |
| 레이아웃 | 좌측: 현재 폴더 이미지 썸네일 스트립 (세로 스크롤) / 우측: 선택 이미지 뷰 |
| 이미지 전환 | 썸네일 클릭 또는 ↑↓ 키로 이전/다음 이미지 |
| 뷰어 종료 | ESC 키 또는 뒤로가기 버튼으로 브라우저 모드 복귀 |
| 이미지 맞춤 | 초기 표시: 뷰 영역에 fit (aspect ratio 유지) |

### F5. 확대/축소

| 항목 | 설명 |
|------|------|
| 확대 | Cmd+ +, 트랙패드 핀치, 또는 툴바 버튼 |
| 축소 | Cmd+ -, 트랙패드 핀치, 또는 툴바 버튼 |
| 원본 크기 | Cmd+0 — 100% 표시 |
| 맞춤 | Cmd+9 — 뷰 영역에 fit |
| 범위 | 10% ~ 500% |
| 패닝 | 확대 상태에서 드래그 또는 스크롤로 이동 |

### F6. 다른 이름으로 저장 (포맷 변환)

| 항목 | 설명 |
|------|------|
| 진입 | 뷰어 또는 브라우저에서 컨텍스트 메뉴 → "다른 이름으로 저장" / Cmd+Shift+S |
| 대화상자 | NSSavePanel에 포맷 선택 드롭다운 추가 |
| 지원 포맷 | JPEG, PNG, WebP, HEIC, HEIF, AVIF, TIFF, BMP, GIF |
| 기본값 | 원본 이미지와 동일 포맷 |
| 품질 설정 | 손실 압축 포맷(JPEG, WebP, HEIC)의 경우 품질 슬라이더(1–100, 기본 85) 표시 |

### F7. 이미지 삭제

| 항목 | 설명 |
|------|------|
| 진입 | 선택 이미지 → Delete 키 또는 컨텍스트 메뉴 → "삭제" |
| 확인 | 삭제 전 확인 다이얼로그 표시 ("이미지 N개를 휴지통으로 이동하시겠습니까?") |
| 동작 | 영구 삭제가 아닌 macOS 휴지통으로 이동 (NSWorkspace.shared.recycle) |
| 뷰어 | 삭제 후 다음 이미지 자동 선택. 마지막 이미지인 경우 브라우저 모드로 복귀 |

### F8. 이미지 이동

| 항목 | 설명 |
|------|------|
| 드래그 앤 드롭 | 브라우저에서 이미지를 사이드바 폴더에 드롭 |
| 이동 표시 | 드래그 시 이동 아이콘, 드롭 대상 폴더 하이라이트 |
| 확인 | 드롭 즉시 이동 (확인 다이얼로그 없음) |
| 실패 처리 | 권한 부족 또는 동일 이름 파일 존재 시 알림 표시 |
| 컨텍스트 메뉴 | "이동" 선택 시 폴더 선택 패널(NSOpenPanel) 표시 |

### F9. 이미지 복사

| 항목 | 설명 |
|------|------|
| 컨텍스트 메뉴 | "복사" 선택 시 대상 폴더 선택 패널(NSOpenPanel) 표시 |
| 동일 이름 | 대상 폴더에 동일 이름 파일 존재 시 "이름 변경 / 덮어쓰기 / 건너뛰기" 선택 |

### F10. 파일 이름 변경

| 항목 | 설명 |
|------|------|
| 진입 | 브라우저에서 이미지 선택 후 Enter 키, 또는 컨텍스트 메뉴 → "이름 변경" |
| 동작 | 파일 이름 부분이 인라인 편집 모드로 전환 (확장자 제외한 이름 부분만 선택) |
| 확정 | Enter 키 또는 포커스 아웃 시 이름 변경 실행 |
| 취소 | ESC 키로 편집 취소 (원래 이름 복원) |
| 유효성 검사 | 파일 시스템에서 허용하지 않는 문자(/, :) 입력 시 인라인 에러 표시 |
| 빈 이름 | 이름을 비워둔 채 확정 시 원래 이름 복원 |
| 동일 이름 | 같은 폴더에 동일 이름 파일이 이미 있으면 에러 Alert 표시 ("같은 이름의 파일이 이미 존재합니다") |
| 확장자 보호 | 확장자 변경 시 확인 다이얼로그 표시 ("확장자를 변경하면 파일을 열 수 없을 수 있습니다. 변경하시겠습니까?") |
| 뷰어 | 뷰어 모드에서는 컨텍스트 메뉴를 통해 이름 변경 가능 — 파일 이름 변경 후 뷰어 타이틀 즉시 반영 |

## 8. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 성능 | 1,000개 이미지 폴더에서 썸네일 로딩 시 스크롤 끊김 없음 (비동기 + 캐시) |

| 접근성 | VoiceOver 지원 (이미지 이름, 폴더 이름 읽기) |
| 호환성 | macOS 14 (Sonoma) 이상 지원 |
| 보안 | 앱 샌드박스 적용, 사용자가 명시적으로 선택한 폴더만 접근 |

## 9. 정보 구조 / 사용자 여정

### 전체 레이아웃

```
┌──────────────────────────────────────────────────┐
│  Toolbar (뒤로가기, 확대/축소, 메뉴)              │
├──────────┬───────────────────────────────────────┤
│          │                                       │
│ Sidebar  │         Content Area                  │
│ (폴더    │                                       │
│  트리)   │  [브라우저 모드]  폴더 + 이미지 그리드  │
│          │                                       │
│          │  [뷰어 모드]  썸네일 스트립 | 이미지 뷰  │
│          │                                       │
├──────────┴───────────────────────────────────────┤
│  Status Bar (파일 수, 선택 정보)                   │
└──────────────────────────────────────────────────┘
```

### 사용자 여정

```
앱 실행
  → 루트 폴더 선택 (최초 실행 시)
  → 사이드바에 폴더 트리 로드
  → 폴더 선택
  → 브라우저에 폴더/이미지 목록 표시
  → 이미지 더블 클릭
  → 뷰어 모드 진입 (썸네일 스트립 + 이미지 뷰)
  → 확대/축소, 이전/다음 탐색
  → ESC로 브라우저 복귀
```

## 10. 핵심 사용자 플로우

### Flow 1: 폴더 탐색 → 이미지 뷰잉

```
[사이드바 폴더 클릭]
  → 브라우저에 하위 폴더 + 이미지 표시
  → [이미지 더블 클릭]
  → 뷰어 모드 전환 (좌: 썸네일 스트립 / 우: 이미지 뷰)
  → [썸네일 클릭 또는 ↑↓] → 이미지 전환
  → [ESC] → 브라우저 모드 복귀
```

### Flow 2: 이미지 이동 (드래그 앤 드롭)

```
[브라우저에서 이미지 선택]
  → 드래그 시작
  → 사이드바 폴더 위에 호버 → 폴더 하이라이트
  → 드롭
  → 이미지가 대상 폴더로 이동
  → 브라우저에서 해당 이미지 제거
```

### Flow 3: 포맷 변환 저장

```
[뷰어 또는 브라우저에서 이미지 선택]
  → Cmd+Shift+S 또는 컨텍스트 메뉴 → "다른 이름으로 저장"
  → NSSavePanel 표시 (포맷 선택 드롭다운 + 품질 슬라이더)
  → 포맷 선택, 파일 이름 입력
  → "저장" 클릭
  → 변환 완료
```

### Flow 4: 이미지 삭제

```
[브라우저 또는 뷰어에서 이미지 선택]
  → Delete 키 또는 컨텍스트 메뉴 → "삭제"
  → 확인 다이얼로그 표시
  → "삭제" 확인
  → 이미지가 휴지통으로 이동
  → 브라우저: 목록에서 제거 / 뷰어: 다음 이미지로 전환
```

## 12. 화면 상세 스펙

### SCR-01: 메인 윈도우 — 브라우저 모드

| 항목 | 내용 |
|------|------|
| 목적 | 폴더 구조를 탐색하며 이미지를 썸네일로 브라우징 |
| 진입 조건 | 앱 실행 시 (기본 모드) 또는 뷰어에서 ESC |
| 사용자 목표 | 원하는 폴더의 이미지를 빠르게 훑어보기 |
| 주요 콘텐츠 | 좌측 사이드바(폴더 트리), 우측 콘텐츠 영역(폴더+이미지 그리드) |
| Primary CTA | 이미지 더블 클릭 → 뷰어 모드 |
| Secondary 액션 | 폴더 이동, 다중 선택, 컨텍스트 메뉴(이름 변경/복사/이동/삭제/저장) |
| 입력 요소 | 없음 |
| **상태** | |
| - default | 사이드바에 폴더 트리, 우측에 선택 폴더 내용 표시 |
| - loading | 대용량 폴더: 썸네일 placeholder 표시 후 비동기 로딩 |
| - empty | 폴더에 이미지/하위 폴더 없음 → "이 폴더는 비어 있습니다" 메시지 |

| - no permission | 샌드박스 미허용 폴더 → 폴더 선택 안내 메시지 |
| 성공 → 다음 화면 | 이미지 더블 클릭 → SCR-02 |
| 실패 → 복구 | 폴더 접근 실패 시 이전 폴더 유지 |

## 14. 인수 조건 (Acceptance Criteria)

### AC-01: 폴더 트리 탐색

```
Scenario 1 — 폴더 선택
  Given 사이드바에 폴더 트리가 표시되어 있다
  When 사용자가 폴더를 클릭한다
  Then 우측 브라우저에 해당 폴더의 하위 폴더와 이미지가 표시된다

Scenario 2 — 폴더 펼침/접기
  Given 하위 폴더가 있는 폴더가 있다
  When disclosure triangle을 클릭한다
  Then 하위 폴더가 펼쳐지거나 접힌다

Scenario 3 — 접근 불가 폴더
  Given 읽기 권한이 없는 폴더가 있다
  When 해당 폴더를 클릭한다
  Then "이 폴더에 접근할 수 없습니다" 메시지가 표시된다
```

### AC-02: 이미지 브라우징

```
Scenario 1 — 이미지 목록 표시
  Given 사이드바에서 이미지가 포함된 폴더를 선택했다
  When 브라우저가 로드된다
  Then 하위 폴더가 먼저, 이미지가 다음으로 표시된다
  And 이미지는 썸네일과 파일 이름으로 표시된다

Scenario 2 — 빈 폴더
  Given 하위 폴더와 이미지가 없는 폴더를 선택했다
  When 브라우저가 로드된다
  Then "이 폴더는 비어 있습니다" 메시지가 표시된다

Scenario 3 — 대용량 폴더 (1,000개 이상)
  Given 이미지 1,000개 이상이 있는 폴더를 선택했다
  When 브라우저가 로드된다
  Then 썸네일이 비동기로 로드되며 스크롤이 끊기지 않는다
```

### AC-03: 이미지 뷰어

```
Scenario 1 — 뷰어 진입
  Given 브라우저에 이미지가 표시되어 있다
  When 이미지를 더블 클릭한다
  Then 우측 영역이 뷰어 모드로 전환된다
  And 좌측에 썸네일 스트립, 우측에 이미지가 표시된다

Scenario 2 — 이미지 전환
  Given 뷰어 모드가 활성화되어 있다
  When 썸네일 스트립에서 다른 이미지를 클릭한다
  Then 우측 이미지 뷰가 해당 이미지로 전환된다

Scenario 3 — 뷰어 종료
  Given 뷰어 모드가 활성화되어 있다
  When ESC 키를 누른다
  Then 브라우저 모드로 복귀한다
  And 직전에 보던 이미지가 브라우저에서 선택 상태로 표시된다

Scenario 4 — 손상된 이미지
  Given 손상된 이미지 파일이 있다
  When 해당 이미지를 뷰어에서 열려고 한다
  Then "이미지를 표시할 수 없습니다" 에러 메시지가 표시된다
  And 앱이 크래시되지 않는다
```

### AC-04: 확대/축소

```
Scenario 1 — 확대
  Given 뷰어에 이미지가 표시되어 있다
  When Cmd+ + 를 누른다
  Then 이미지가 확대된다

Scenario 2 — 축소
  Given 뷰어에 이미지가 확대된 상태다
  When Cmd+ - 를 누른다
  Then 이미지가 축소된다

Scenario 3 — 범위 제한
  Given 이미지가 500%로 확대되어 있다
  When 추가로 확대를 시도한다
  Then 더 이상 확대되지 않는다 (최대 500%)

Scenario 4 — 패닝
  Given 이미지가 뷰 영역보다 크게 확대되어 있다
  When 드래그한다
  Then 이미지가 드래그 방향으로 이동한다
```

### AC-05: 다른 이름으로 저장 (포맷 변환)

```
Scenario 1 — 포맷 변환 저장
  Given 뷰어에 PNG 이미지가 표시되어 있다
  When Cmd+Shift+S → 포맷을 JPEG로 선택 → "저장" 클릭
  Then JPEG 포맷으로 파일이 저장된다

Scenario 2 — 품질 설정
  Given 다른 이름으로 저장 패널에서 JPEG 포맷을 선택했다
  When 품질 슬라이더가 표시된다
  Then 슬라이더로 1–100 사이 품질을 선택할 수 있다

Scenario 3 — 저장 실패
  Given 쓰기 권한이 없는 폴더를 선택했다
  When "저장"을 클릭한다
  Then 에러 Alert가 표시된다 ("파일을 저장할 수 없습니다: 권한 부족")
```

### AC-06: 이미지 삭제

```
Scenario 1 — 삭제 확인
  Given 브라우저에서 이미지 2개를 선택했다
  When Delete 키를 누른다
  Then "이미지 2개를 휴지통으로 이동하시겠습니까?" 확인 다이얼로그가 표시된다

Scenario 2 — 삭제 실행
  Given 삭제 확인 다이얼로그가 표시되어 있다
  When "삭제"를 클릭한다
  Then 이미지가 macOS 휴지통으로 이동된다
  And 브라우저 목록에서 해당 이미지가 제거된다

Scenario 3 — 삭제 취소
  Given 삭제 확인 다이얼로그가 표시되어 있다
  When "취소"를 클릭한다
  Then 아무 일도 일어나지 않고 다이얼로그가 닫힌다
```

### AC-07: 이미지 이동 (드래그 앤 드롭)

```
Scenario 1 — 드래그 앤 드롭 이동
  Given 브라우저에서 이미지를 선택하여 드래그 중이다
  When 사이드바의 다른 폴더 위에 드롭한다
  Then 이미지가 해당 폴더로 이동된다
  And 브라우저 목록에서 해당 이미지가 제거된다

Scenario 2 — 드롭 대상 하이라이트
  Given 이미지를 드래그 중이다
  When 사이드바 폴더 위에 호버한다
  Then 해당 폴더가 하이라이트 표시된다

Scenario 3 — 이동 실패 (동일 이름)
  Given 대상 폴더에 동일 이름의 파일이 존재한다
  When 이미지를 드롭한다
  Then 파일 충돌 해결 다이얼로그(DLG-04)가 표시된다
```

### AC-08: 파일 이름 변경

```
Scenario 1 — 인라인 이름 변경
  Given 브라우저에서 이미지가 선택되어 있다
  When Enter 키를 누른다
  Then 파일 이름이 인라인 편집 모드로 전환된다
  And 확장자를 제외한 이름 부분만 선택된다

Scenario 2 — 이름 변경 확정
  Given 인라인 편집 모드에서 새 이름을 입력했다
  When Enter 키를 누른다
  Then 파일 이름이 변경된다
  And 브라우저에서 변경된 이름으로 표시된다

Scenario 3 — 이름 변경 취소
  Given 인라인 편집 모드가 활성화되어 있다
  When ESC 키를 누른다
  Then 원래 이름으로 복원된다

Scenario 4 — 동일 이름 충돌
  Given 같은 폴더에 "photo.jpg"가 이미 존재한다
  When 다른 이미지의 이름을 "photo.jpg"로 변경하려 한다
  Then "같은 이름의 파일이 이미 존재합니다" 에러 Alert가 표시된다
  And 인라인 편집 모드가 유지된다

Scenario 5 — 확장자 변경 경고
  Given 인라인 편집 모드에서 확장자를 변경했다
  When Enter 키를 누른다
  Then "확장자를 변경하면 파일을 열 수 없을 수 있습니다" 확인 다이얼로그가 표시된다

Scenario 6 — 빈 이름
  Given 인라인 편집 모드에서 이름을 모두 지웠다
  When Enter 키를 누른다
  Then 원래 이름으로 복원된다

Scenario 7 — 잘못된 문자
  Given 인라인 편집 모드가 활성화되어 있다
  When 파일 이름에 "/" 또는 ":" 를 입력한다
  Then 인라인 에러가 표시된다
```

### AC-09: 이미지 복사

Scenario 1 — 복사
  Given 브라우저에서 이미지를 선택했다

Scenario 2 — 동일 이름 충돌
  Given 대상 폴더에 동일 이름 파일이 있다
  When 복사를 실행한다
  Then "이름 변경 / 덮어쓰기 / 건너뛰기" 선택 다이얼로그가 표시된다
